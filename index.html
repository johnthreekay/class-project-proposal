Application Overview

Ryokan is a self-hosted anime PVR (Personal Video Recorder) that automates the process of finding, downloading, and importing anime into a Jellyfin media library. Users search for anime by title, and Ryokan pulls metadata from AniList (with MAL via Jikan and Kitsu as fallbacks), then searches Nyaa for available releases. A scoring engine ranks results across nine quality tiers from WEB 480p through BD Remux 1080p, and users can send releases directly to qBittorrent with one click. An RSS auto-sync system monitors for new episodes and quality upgrades automatically.

The application tracks a user's anime library with Sonarr-style monitoring modes (all, future, missing, existing, or none) and caches all metadata locally so pages load instantly after initial setup. Ryokan also exposes Sonarr and Radarr v3 API compatibility layers, allowing Seerr to request anime series and movies through it as if it were a native Sonarr/Radarr instance.

I built this because Sonarr doesn't work well for anime. Its RSS sync handles currently airing shows fine, but downloading season batches of finished shows almost always hangs the interactive search because Sonarr uses SXEXX-style episode identifiers that don't match how most anime torrents are named.

Technology Overview

Rust (language), Axum 0.8 (web framework), Tokio (async runtime), SQLite via sqlx (database, runtime queries), Askama (HTML templating, compiled into the binary), reqwest (HTTP client), scraper (HTML parsing for Nyaa), bcrypt (password hashing), serde/serde_json (serialization). Single CSS file with no frontend build step. Containerized with a multi-stage Dockerfile and Docker Compose, self-hosted on a Linux server behind a Cloudflare Tunnel. CI via GitHub Actions (cargo build + cargo test on pushes/PRs to main).

Feature List

  • AniList Search & Metadata with Fallbacks: Search for anime by title via AniList's GraphQL API. MAL (via Jikan) and Kitsu serve as fallback metadata sources when AniList is unavailable. Detail pages show cover art, synopsis, airing status, episode count, and genres.
  • Nyaa Torrent Search & Scoring: Scrape Nyaa for available releases and rank them across nine quality tiers (WEB 480p through BD Remux 1080p) using a multi-factor scoring algorithm based on seeders, downloads, file size, trusted status, batch detection, Blu-ray source bonuses, and preferred release groups.
  • qBittorrent Download Integration: Send torrents to qBittorrent via its Web API with automatic session re-authentication on cookie expiry and one-click grab from search results.
  • RSS Auto-Sync: Automatic polling for new episodes of tracked series with auto-grab based on quality profile thresholds. Handles quality upgrades when better encodes become available.
  • Library with Monitoring Modes: Visual cover art grid of tracked anime with Sonarr-style monitoring modes: all, future, missing, existing, or none. Local metadata caching for instant page loads.
  • Seerr Integration: Sonarr and Radarr v3 API compatibility layers so Seerr can request anime series and movies through Ryokan as if it were a native Sonarr/Radarr instance.
  • Session-Based Authentication: First-run setup flow to create an admin account. Login with bcrypt-hashed passwords, session tokens stored in SQLite, and auth middleware protecting all routes.
  • Settings & Configuration: Web UI for configuring external service connections (qBittorrent, Jellyfin), quality profiles and cutoffs, preferred/blocked release groups, media root path, and title language preference.
  • Post-Processing Pipeline: Moves and renames completed downloads into the media library with Jellyfin library refresh on import. Content-addressed artwork caching.
  • Help System: In-app help page describing features, service configuration, and a quick-start guide.

Future Feature List

  • Source Classification Pipeline: A multi-layer system for tagging releases by source (BluRay, DVD, Web, HDTV) and resolution, replacing the current filename-only detection. Includes filename token parsing, release group identity tables, Nyaa description scraping, temporal inference from airing status, ffprobe stream analysis of downloaded files, and directory structure scanning for BD-exclusive content like creditless OP/EDs and extras.
  • Post-Download Classification: Run ffprobe on imported files to classify source from audio codecs (FLAC/TrueHD = BluRay, AAC = Web), subtitle formats (PGS = BluRay), and container metadata. Handles externally-imported libraries as a first-class case.
  • Manual Override & Needs-Review UI: Users can force-tag any release's source, and a "needs review" list surfaces files where classification signals conflict or are too weak. Manual overrides feed back into the group identity table as suggestions.
  • Per-Series Upgrade Controls: Toggle quality upgrades on a per-series basis rather than globally.
  • Episode-Level Tracking: Track individual episode download status per series, detect collection gaps, and search for missing episodes automatically.
  • Notification System: Discord webhooks for completed downloads, new releases matching tracked series, and failed grabs.

Basic Technical Features

  • HTTP Server & Routing: Axum web server with route handlers, middleware chains, and static file serving.
  • Server-Side Templating: Askama templates with base layout inheritance and dynamic data rendering, compiled into the binary.
  • Database CRUD: SQLite with sqlx for users, sessions, config, library entries, and scheduled task tracking. Idempotent migrations in code.
  • User Authentication & Sessions: Login/logout with bcrypt hashing, session tokens, cookie auth, and route-gating middleware. First-run setup flow.
  • Form Handling & Validation: Settings, setup, and configuration pages with server-side input validation and error feedback.
  • Docker Containerization: Multi-stage Dockerfile (Rust build stage, minimal Debian runtime) and Docker Compose with persistent volumes.
  • CI Pipeline: GitHub Actions workflow running cargo build and cargo test on pushes and pull requests to main.

Advanced Technical Features

  • Multi-Source Metadata with Fallback Chain: AniList GraphQL as the primary metadata provider, with automatic fallback to MAL (via Jikan REST API) and Kitsu when AniList returns errors or rate limits. Three different API formats (GraphQL, REST, JSON:API) unified into a single internal data model.
  • Nyaa HTML Scraping & Scoring Engine: Scrapes and parses Nyaa search result HTML into structured data. Multi-factor scoring algorithm evaluates seeders, downloads, size, encoding, trusted status, batch detection, BD bonuses, and preferred groups across nine quality tiers to produce composite rankings for automated grab decisions.
  • Sonarr/Radarr v3 API Compatibility Layer: Implements enough of the Sonarr and Radarr v3 REST APIs that Seerr treats Ryokan as a native instance, enabling anime request workflows through an existing request management UI without any Seerr modifications.
  • Background Task System & Service Integration: Tokio-spawned background loops for RSS sync, metadata refresh, log cleanup, and post-processing, each reporting status to a scheduled task tracking table. Full qBittorrent Web API integration with automatic session re-authentication on cookie expiry. Jellyfin API integration for library refresh on import.
